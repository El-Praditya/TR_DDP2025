### Persyaratan Teknis

1. **Struktur Program**
   - Array = menyimpan data
   - Implementasi fungsi-fungsi dasar yang diperlukan
   - Proper error handling untuk setiap operasi
   - Implementasi algoritma searching sederhana

2. **Manajemen Data**
   - Penggunaan array untuk penyimpanan
   - Validasi input untuk setiap data
   - Penanganan duplikasi data
   - Backup data sementara dalam program

3. **Keamanan Data**
   - Validasi tipe data untuk setiap input
   - Pencegahan integer overflow
   - Pengecekan batas array
   - Validasi string input (panjang, karakter khusus)

4. **User Interface**
   - Tampilan menu yang rapi dan konsisten
   - Pesan error yang informatif
   - Loading screen sederhana
   - Clear screen di tempat yang tepat

5. **Validasi Input**
   - Validasi semua input user
   - Handling untuk input yang tidak valid
   - Pesan error yang jelas

6. **Bonus Points**
   - Implementasi fitur tambahan yang kreatif
   - Penggunaan array yang efisien
   - Animasi atau efek visual tambahan
   - Error handling yang komprehensif


### Kriteria Penilaian

1. **Fungsionalitas (40%)**
   - Semua fitur wajib berjalan sesuai deskripsi tanpa menggunakan struct atau fitur data kompleks.
   - Implementasi menggunakan array dan fungsi dasar.
   - Operasi input/output dan logika perhitungan (subtotal, pajak, split bill, BMI, dll.) akurat.
   - Validasi input dan penanganan error yang komprehensif.
   - Game dan pola grafis bekerja sesuai spesifikasi.

2. **Kode dan Struktur (30%)**
   - Organisasi kode yang rapi dan modular (fungsi terpisah untuk setiap fitur).
   - Pemanfaatan array dan tipe data dasar secara efektif â€” dilarang penggunaan struct/kelas/vector.
   - Efisiensi algoritma dan penggunaan memori yang sesuai batas tugas.
   - Dokumentasi kode (komentar) dan penamaan variabel yang jelas.

3. **User Interface (20%)**
   - Tampilan menu dan output rapi, konsisten, dan mudah dinavigasi.
   - Pesan error dan instruksi input jelas bagi pengguna.
   - Penggunaan efek sederhana (loading, warna) yang tidak mengganggu kegunaan.

4. **Kreativitas dan Bonus (10%)**
   - Fitur tambahan yang berguna dan sesuai batasan (mis. animasi berbasis karakter, opsi tambahan pada menu).
   - Penggunaan array secara kreatif dan elegan.
   - Perbaikan UX (mis. input safe handling, konfirmasi, struk rapi).
